A little bit about what we do

Developing search engines
that help organisations discover
the hidden value of vast data stores
and uncover their true potential

Mindmaps focuses on integrating large datasets through knowledge ontologies using graph technologies and performing parallelised semantic search on Big Data. The end result is a search engine that can infer meaningful relationships between previously distinct and isolated pieces of information. By leveraging these relationships our search engine can then perform an explicit search over large amounts of data by leveraging these relationships. We can assist organisations in a variety of fields and industries to find information and explore ideas in an integrative manner. Our search engine integrates five distinct fields in computer science: ontology engineering, machine learning, natural language processing, semantic search and distributed computing. Our system architecture is designed with a high degree of modularity and scalability such that it can be easily customised to function with different types of data and industries.

Our first application will be a semantic search engine aimed at the film industry that is capable of traversing the corpus of movie information on the internet to allow users to perform more nuanced search and provide more meaningful search results.

Technology that we use

These are the main technologies
we use throughout our workflow,
but there are more
that we continue to explore

  • Java

    All of our backend systems are developed in one language

  • Spring

    Spring framework forms the backbone of our web services

  • CoreNLP

    Natural sentence queries are parsed through an engine built using CoreNLP

  • Apache Storm

    Computations are distributed to a cluster developed using Storm

  • Apache Spark

    Data crunches for Machine Learning models are parallelised using Spark

  • Apache Cassandra

    We store our preprocessing data in scalable, NoSQL Cassandra

  • TitanDB

    Our distributed semantic graph data is persisted using TitanDB

  • Gremlin / TinkerPop3

    Gremlin and the TinkerPop3 stack allows us to perform advanced graph computing
